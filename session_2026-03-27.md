#### Discovery Findings

**Installed versions:** react 19.2.3, next 15.4.6, next-themes 0.3.0, @radix-ui/react-dropdown-menu 2.1.1, @radix-ui/react-primitive 2.0.0

**Bug 1 — Theme Toggler:**
- `next-themes@0.3.0` was built for React 18. React 19.2.3 is installed.
- `setTheme` from `useTheme()` likely non-functional due to React 19 incompatibility.
- ThemeProvider.tsx imports types from `next-themes/dist/types` — valid for 0.3.0, but path removed in 0.4.x.
- `"use client"`, `suppressHydrationWarning`, ThemeProvider config all correct — not the cause.
- Fix: Upgrade `next-themes` to `^0.4.4`, update import path in ThemeProvider.tsx.

**Bug 2 — Auth/Logout:**
- Navbar.tsx and NavbarHome.tsx render the avatar dropdown (with Logout) UNCONDITIONALLY — no `{user && ...}` guard. This is why Logout shows when not authenticated.
- Logout.tsx renders `<div onClick={handleLogout}>` INSIDE `DropdownMenuItem`. Radix captures pointer events at the Item level — inner div's `onClick` likely never fires.
- Double-call: Logout.tsx calls `/api/auth/logout` via fetch, then calls `useAuthStore.getState().logout()` which calls `/api/auth/logout` AGAIN.
- Zustand persist stores `isAuthenticated` in localStorage, but Navbar doesn't use Zustand — it uses `supabase.auth.getUser()` directly. Stale Zustand state is NOT causing the visible-Logout issue.
- Fix: (A) Conditionally render dropdown based on user state, (B) Move logout handler to DropdownMenuItem onSelect, (C) Remove duplicate fetch call.
